Introdução

Este tem como objetivo exemplificar um código escrito em C.

O código em questão, abre o um link no Google Chrome na tela estendida em full-screen, e reinicia o aplicativo a cada 30 minutos.

Comentários

Comentários são partes importantes dentro do código. Na linguagem C, costuma-se começar o código com um comentário que descreve o programa, mostra a versão do aplicativo e também o autor.

Bibliotecas

Bibliotecas são partes externas do código, as funções requerem bibliotecas especificas para serem executadas corretamente.

São adicionadas com o parâmetro "#include".

As bibliotecas adicionadas a este código são:

  • #include <stdio.h>
  • #include <stdlib.h>
  • #include <time.h>
  • #include <string.h>
  • #include <unistd.h>
Variáveis

Variável é um local reservado na memoria para armazenar um tipo de dado.

As variáveis adicionadas a este código são:
int hora=0; int minuto=0; int segundo=0; int diadasemana=0; int diadomes=0; int mes=0; int ano=0; int resp=0; unsigned int mSeconds=600000; char command[80];

Protótipo das funções

Os Protótipos de funções declaram as funções que serão usadas no programa.

Função principal

A função "main" serve como ponto de partida para a execução do programa.

É dentro dela que as entradas dos programas são reconhecidas, processadas e uma saída é gerada.
Além disso, ela também é responsável por encerrar o programa informando se foi ou não uma execução bem-sucedida a partir do retorno de um código.

Tratamento das funções

Conjunto de comandos agrupados em um bloco, que recebe um nome e através deste pode ser evocado.

As funções servem para:

  • Para permitir o reaproveitamento de código já construído(por você ou por outros programadores.
  • Para evitar que um trecho de código que seja repetido várias vezes dentro de um mesmo programa.
  • Para permitir a alteração de um trecho de código de uma forma mais rápida. Com o uso de uma função é preciso alterar apenas dentro da função que se deseja.
  • Para que os blocos do programa não fiquem grandes demais e, por consequência, mais difíceis de entender.
  • Para facilitar a leitura do programa-fonte de uma forma mais fácil.
  • Para separar o programa em partes(blocos) que possam ser logicamente compreendidos de forma isolada.
